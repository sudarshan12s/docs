<Tip>
**Compatibility**: Only available on Node.js.
</Tip>

Oracle AI Vector Search stores embeddings alongside relational data inside Oracle Database. The `@langchain/oracle` package provides the `OracleVS` vector store plus helpers for loading, summarizing, and embedding documents.

## Overview

### Integration details

| Class | Package | Hybrid search | PY support |
| :--- | :--- | :---: | :---: |
| [`OracleVS`](https://api.js.langchain.com/classes/langchain_oracle.OracleVS.html) | [`@langchain/oracle`](https://www.npmjs.com/package/@langchain/oracle) | ✅ | ✅ |

## Setup

Install Oracle client bindings and the LangChain Oracle helpers:

<CodeGroup>
```bash npm
npm install @langchain/oracle oracledb @langchain/core
```
```bash yarn
yarn add @langchain/oracle oracledb @langchain/core
```
```bash pnpm
pnpm add @langchain/oracle oracledb @langchain/core
```
</CodeGroup>

Set connection credentials for the Oracle user that will own the vector table:

```bash
export ORACLE_USER=testuser
export ORACLE_PASSWORD=testuser
export ORACLE_DSN="localhost:1521/free"
```

## Create a vector store

The example below assumes that you already created a table with vector and metadata columns and generated embeddings using [`OracleEmbeddings`](/oss/javascript/integrations/text_embedding/oracleai).

:::js
```typescript
import oracledb from "oracledb";
import { OracleVS } from "@langchain/oracle";
import { Document } from "@langchain/core/documents";

const connection = await oracledb.getConnection({
  user: process.env.ORACLE_USER,
  password: process.env.ORACLE_PASSWORD,
  connectionString: process.env.ORACLE_DSN,
});

const vectorStore = new OracleVS(connection, {
  owner: "TESTUSER",
  tablename: "DEMO_VECTORS",
  idColumn: "ID",
  embeddingColumn: "EMBEDDING",
  metadataColumns: ["DOC_ID", "TITLE"],
});

const docs: Document[] = [
  {
    pageContent: "LangChain works great with Oracle Database.",
    metadata: { doc_id: "doc-1", title: "RAG overview" },
  },
];

await vectorStore.addDocuments(docs);
```
:::

## Query with hybrid filters

Combine semantic similarity with SQL predicates on the same table.

:::js
```typescript
const results = await vectorStore.similaritySearch("oracle rag", 3, {
  where: "DOC_ID LIKE 'doc-%'",
});

console.log(results[0]?.pageContent);
```
:::
