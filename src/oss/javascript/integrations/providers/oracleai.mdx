**Compatibility**: Only available on Node.js.
</Tip>

Oracle AI Vector Search keeps structured data, documents, and embeddings inside Oracle Database. The `@oracle/langchain-oracledb` package exposes LangChain components for loading content, chunking, generating embeddings, running in-database summarization, and storing vectors—so you can build retrieval-augmented generation (RAG) pipelines without moving data across services.

## Why Oracle AI Vector Search

- Run semantic and relational queries on the same Oracle table
- Keep sensitive content inside the database by running ONNX models locally
- Swap in managed providers (OCI Generative AI, Hugging Face) with a configuration change
- Use Oracle-native features—transactions, partitioning, RAC, Data Guard—for vector workloads

## Install and configure

### Packages

Install the Oracle helpers and ensure your environment meets the [`oracledb`](https://www.npmjs.com/package/oracledb) driver prerequisites.

<CodeGroup>
```bash npm
npm install @oracle/langchain-oracledb @langchain/core
```
```bash yarn
yarn add @oracle/langchain-oracledb @langchain/core
```
```bash pnpm
pnpm add @oracle/langchain-oracledb @langchain/core
```
</CodeGroup>

### Credentials

Configure connection details for the Oracle user that owns your vector tables and credentials.

```bash
export ORACLE_USER=testuser
export ORACLE_PASSWORD=testuser
export ORACLE_DSN="localhost:1521/free"
```

If you plan to call managed providers (OCI Generative AI, Hugging Face), create Oracle credentials (for example `OCI_CRED`, `HF_CRED`) using the PL/SQL helpers described in the [Oracle AI Vector Search guide](https://www.oracle.com/pls/topic/lookup?ctx=dblatest&id=GUID-EC9DDB58-6A15-4B36-BA66-ECBA20D2CE57).

## Quickstart pipeline

<Steps>
<Step title="Connect to Oracle Database">

```typescript
import oracledb from "oracledb";

const connection = await oracledb.getConnection({
  user: process.env.ORACLE_USER,
  password: process.env.ORACLE_PASSWORD,
  connectionString: process.env.ORACLE_DSN,
});
```
</Step>
<Step title="Load documents">

```typescript
import { OracleDocLoader } from "@oracle/langchain-oracledb";

const loader = new OracleDocLoader(connection, {
  owner: "TESTUSER",
  tablename: "DEMO_TAB",
  colname: "DATA",
});

const docs = await loader.load();
```
</Step>
<Step title="Chunk and embed">

```typescript
import { OracleTextSplitter, OracleEmbeddings } from "@oracle/langchain-oracledb";

const splitter = new OracleTextSplitter(connection, {
  split: "chars",
  max: 500,
  normalize: "all",
});

const embeddings = new OracleEmbeddings(connection, {
  provider: "database",
  model: "DEMO_MODEL",
});

const chunks = await splitter.splitText(docs[0].pageContent);
const vectors = await embeddings.embedDocuments(chunks);
```
</Step>
<Step title="Store and query">

```typescript
import { OracleVS } from "@oracle/langchain-oracledb";

const vectorStore = new OracleVS(connection, {
  owner: "TESTUSER",
  tablename: "DEMO_VECTORS",
  idColumn: "ID",
  embeddingColumn: "EMBEDDING",
  metadataColumns: ["DOC_ID", "TITLE"],
});

await vectorStore.addDocuments(
  chunks.map((pageContent, index) => ({
    pageContent,
    metadata: { doc_id: `doc-${index}` },
  })),
);

const results = await vectorStore.similaritySearch("oracle rag", 3);
```
</Step>
</Steps>

## Provider configuration

- **In-database ONNX**: Upload ONNX files once with `OracleEmbeddings.loadOnnxModel(connection, "/path/to/models", "all-minilm-l12-v2.onnx", "DEMO_MODEL")` to keep workload data inside Oracle.
- **OCI Generative AI**: Set `provider: "ocigenai"` plus `credential_name`, `url`, and `model` fields on @[OracleEmbeddings] or @[OracleSummary].
- **Hugging Face**: Set `provider: "huggingface"` along with `credential_name`, `url`, and model ID. Use the optional proxy argument if outbound traffic goes through an HTTP proxy.
